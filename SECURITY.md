# Security Policy 🛡️

## Supported Versions

Currently, `ez-cpp` is in active development. Only the latest versions are actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.1.x| :white_check_mark: |
| < 0.1.x | :x:                |

## Reporting a Vulnerability

We take security seriously, even for a "vibe" library. If you discover a security vulnerability within `ez-cpp` (e.g., an undiscovered memory leak, buffer overflow in the input stream, or undefined behavior that could be exploited), please **DO NOT** open a public issue.

Instead, please report it directly to the maintainer:
- **Email:** sushilibdev@gmail.com
- **Subject:** [SECURITY] ez-cpp Vulnerability Report

Please include a brief description of the vulnerability and, if possible, a code snippet reproducing the issue. I will acknowledge your report within 48 hours and work on a patch immediately. 

Thank you for helping keep `ez-cpp` safe for everyone!
